OPTION 1: GITHUB BACKUP (RECOMMENDED)
--------------------------------------

Why: Free, cloud-based, version controlled, accessible anywhere

Setup Time: 5 minutes

Steps:
1. Create GitHub account at https://github.com (if you don't have one)
2. Go to https://github.com/new to create a new repository
3. Name it: triple-waza-challenge (or any name you prefer)
4. Set to PRIVATE (important for security!)
5. Do NOT initialize with README
6. Click "Create repository"
7. Copy the commands GitHub shows you

Then run these commands:

   git remote add origin https://github.com/YOUR_USERNAME/triple-waza-challenge.git
   git push -u origin main

That's it! Your entire site is now backed up to GitHub cloud.

Result: Entire site backed up to GitHub, accessible from anywhere

WHAT DOES .md MEAN?
================================================================================

.md = Markdown

Markdown is a simple text formatting language that makes text easier to read.

Example:
- **bold text** displays as bold
- # Heading creates a large heading
- - List item creates bullet points
- [link](url) creates clickable links

Why we use .md files:
✓ Easy to read even in plain text
✓ Looks nice on GitHub
✓ Can be opened in any text editor
✓ Converts nicely to HTML/PDF if needed

You can read .md files in:
- Any text editor (Notepad, TextEdit, etc.)
- GitHub (displays formatted automatically)
- VS Code (with preview)
- Online viewers like: dillinger.io

This .txt file contains the same information without markdown formatting,
making it easier to read in basic text editors.
